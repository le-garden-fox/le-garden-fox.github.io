Grocking TCP Network Connections

I think an indicator the marks the difference of a mid level and a senior level engineer is their understanding of the computer network. The two main underlyning protocols of that system are TCP and UDP. Each of those protocols has its place and use cases, but TCP is perhaps the most used when writing programms. In this article I want to explain why it is used for the majority of communications, how to debug it and illuminate it. If you are any kind of software engineer, frontend, backend, or data you use it every day. Understand it will strengthen your skills and become more independent vs relying on a DevOps engineer or Stack Overflow.

So to begin I want to explain the fundamentals of Transmission Control Protocol (TCP). Here are some of the proeprites of the protocol:

  • A TCP connection is validated and confirmed before data is trasnsmitted
  • Connections can be reused to transmit data
  • TCP is a stream protocol
  • Data sent over TCP to a destination is sent in packets
    • packets are fixed size collections of bits.
    • packets have several sections
      • headers
      • body
    • Packets can be sent out of order to the destination
      • The packets are reorganized at the destination in the correct order
      • the destination can perform data inegrity/ error checking on each packet
        • the dstination can request a packet be resent when it finds an error in a packet
